# Phase 3 вЂ” Code Generation & Live Preview

**Goal:** Transform the node graph built in Phase 2 into real Papyrus source code (`.psc`) that can be previewed in real time and compiled to `.pex` bytecode. Introduce Properties and User-Defined Functions as first-class script-level constructs. The code generator must handle all В§11 edge cases gracefully and pass a lint pass before compilation.

**Exit criteria (M3):** Placing an `OnInit` node в†’ `Debug.Notification` node в†’ connecting them в†’ clicking the canvas produces correct `.psc` in the Preview panel within one frame. Clicking Compile invokes the compiler and produces a `.pex`. All code-generation unit tests pass.

### 3.1 вЂ” Graph Traversal (DFS execution-flow walker) `P0`

> ROADMAP В§53, В§11

- [ ] Create `src/codegen/GraphTraversal.h` / `GraphTraversal.cpp`:
  - `GraphTraversal::Traverse(const ScriptGraph&, uint64_t start_node_id) в†’ std::vector<uint64_t>`
  - DFS along **execution-flow** edges only (Exec pins), starting from a given Event or Function Entry node
  - Returns an **ordered list of node IDs** representing the linear statement sequence
  - Cycle detection: maintain a visited set; if a node is re-visited via exec-flow, halt traversal, record the offending back-edge, continue from the next unvisited branch
  - Dead-end handling: exec-out pin with no connection в†’ traversal ends on that branch
  - `GetCycleEdges() в†’ std::vector<std::pair<uint64_t, uint64_t>>` вЂ” for UI highlighting
- [ ] Collect **data dependencies** for each node in the ordered list:
  - For each data-input pin in the statement list: if connected, recursively resolve the source node's output (depth-first data-flow evaluation)
  - Result: for each statement node, an ordered list of `(pin_name в†’ resolved_value_or_temp_var)` pairs
- [ ] Handle edge cases per ROADMAP В§11:
  - Cycle in exec-flow в†’ emit `; [ERROR: Execution-flow cycle detected at node <id>]`, mark `has_errors = true`
  - Disconnected exec-in node в†’ skip (never reached); mark as "unreachable" for lint
  - Empty graph в†’ return empty list (valid)
  - No Event node в†’ return empty list (generates property-only script)
- [ ] **Unit tests** (`tests/test_codegen.cpp`):
  - Linear chain of 3 nodes в†’ correct order
  - Branch at `If/Else` в†’ both branches captured in correct order (true then false)
  - `While` loop body в†’ body nodes + loop-back edge recognised
  - Cycle в†’ halts + `has_errors = true` + `GetCycleEdges()` non-empty
  - Empty graph в†’ empty result, no crash
- [ ] Acceptance: 5+ tests pass; traversal output is deterministic
